## ✅ 현재 연동 상태

| 항목 | 상태 | 비고 |
|------|------|------|

## 🚀 서버 실행

Git Bash에서:

```bash
cd ~/Desktop/공공데이터공모전/HAESOLOK
npm install
npm run dev
```

브라우저에서 👉 **http://localhost:3000** 접속

> ⚠️ 서버를 켜놓은 채로 아래 테스트 진행

---

## 🔍 테스트 시나리오 (순서대로)

### STEP 1 — 온보딩: 학교 검색 (NEIS 연동 확인)

1. `http://localhost:3000/onboarding` 접속
2. **학교급** 선택 (예: 중학교)
3. **학교 검색** 입력창에 학교명 입력 (예: `남서울`)
4. **검색** 버튼 클릭

**✅ 확인 포인트**
- 검색 결과에 실제 학교 목록이 나오는가?
- 학교 선택 시 초록 체크 뱃지 + 시도교육청명이 표시되는가?

5. 원하는 학교 선택 → **해소록 시작하기** 클릭

---

### STEP 2 — 민원 기록: 학사일정 자동 첨부 (NEIS 연동 확인)

1. `http://localhost:3000/complaints/new` 접속
2. 하단 **"자동 첨부되는 공공데이터 맥락"** 카드 확인

**✅ 확인 포인트**
- "오늘의 학사일정" 옆에 **나이스 실시간** 초록 뱃지가 표시되는가?
- 오늘 실제 학사일정이 나오는가? (일정 없으면 "오늘 등록된 학사 일정 없음")

3. 민원 내용 작성 예시:
   ```
   오늘 수업 중 학부모가 갑자기 교실로 찾아와 수업 방식에 대해
   항의했습니다. 다른 학생들이 보는 앞에서 큰 소리로 이의를 제기했고
   약 10분간 수업이 중단되었습니다.
   ```
4. 발생 장소: `교실`
5. 민원인 유형: `학부모`
6. **AI 대응 가이드 생성하기** 클릭

---

### STEP 3 — AI 가이드 확인 (Gemini 연동 확인)

`http://localhost:3000/ai-guide` 자동 이동

**✅ 확인 포인트**
- 상단 "예시 데이터(Fallback)" 문구가 **없는가?** → 없으면 실제 Gemini 응답
- **민원 유형**이 입력 내용 기반으로 분류되는가?
- **대응 위험도** 표시되는가?
- **대응 원칙** 3~4개가 구체적으로 작성되었는가?
- **답변 초안**이 상황을 반영하여 생성되었는가?
- **참고한 공공데이터 맥락** 태그에 나이스 학사일정이 포함되는가?

---

### STEP 4 — 온도 체크인 테스트

1. `http://localhost:3000/check-in` 접속
2. 오늘의 온도 슬라이더 조절 후 저장

**✅ 확인 포인트**
- 저장 후 대시보드로 이동하는가?
- 대시보드에 오늘의 온도가 반영되는가?

---

### STEP 5 — 주간 리포트 테스트 (Gemini 연동 확인)

1. `http://localhost:3000/report` 접속

**✅ 확인 포인트**
- AI가 주간 분석 요약을 실제로 생성하는가?
- 리포트 내용이 기록된 온도 흐름을 반영하는가?

## 🔧 API 직접 테스트 (Git Bash)

서버가 실행 중인 상태에서 **새 Git Bash 탭**을 열고:

### 학교 검색 API 테스트
```bash
curl -s -X POST http://localhost:3000/api/neis/school-search \
  -H "Content-Type: application/json" \
  -d '{"schoolName":"남서울","schoolLevel":"중학교"}' | python -m json.tool
```

### 학사일정 API 테스트
```bash
curl -s -X POST http://localhost:3000/api/neis/schedule \
  -H "Content-Type: application/json" \
  -d '{"educationOfficeCode":"B10","schoolCode":"7132131"}' | python -m json.tool
```

> python이 없으면 `| python -m json.tool` 부분 제거해도 됨 (결과는 동일)

## 🐛 문제 발생 시 체크리스트

### AI 가이드가 예시 데이터(Fallback)로 나올 때
```
원인 1: .env.local의 LLM_PROVIDER가 mock으로 되어 있음
→ LLM_PROVIDER=gemini 확인

원인 2: 서버를 .env.local 수정 전에 실행함
→ Ctrl+C 로 서버 종료 후 npm run dev 재실행
```

### 학교 검색 결과가 없을 때
```
원인 1: 학교명 2글자 미만 입력
→ 최소 2글자 이상 입력

원인 2: NEIS_API_KEY 오류
→ .env.local의 NEIS_API_KEY 값 확인
```

### 학사일정이 "예시 데이터"로 나올 때
```
원인: 온보딩에서 학교를 선택하지 않음
→ /onboarding 에서 학교 검색 후 반드시 선택
```
