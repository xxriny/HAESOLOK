# HAESOLOK 실행 가이드

이 문서는 로컬 환경에서 HAESOLOK을 실행하고 검증하는 절차를 정리합니다.

## 1. 사전 준비

| 항목 | 권장 버전 | 확인 명령 |
| --- | --- | --- |
| Node.js | 18 이상 | `node -v` |
| npm | 8 이상 | `npm -v` |

## 2. 프로젝트 폴더로 이동

```powershell
cd E:\HAESOLOK
```

다른 위치에 클론했다면 해당 프로젝트 루트로 이동합니다.

## 3. 의존성 설치

```powershell
npm install
```

최초 1회 실행하면 `node_modules`가 생성됩니다.

## 4. 환경변수 설정

프로젝트 루트에 `.env.local` 파일을 생성합니다.

### API 키 없이 실행하는 경우

```env
LLM_PROVIDER=mock
```

이 모드는 외부 AI API를 호출하지 않고 예시 데이터를 사용합니다. 빠른 시연이나 UI 확인에 적합합니다.

### Gemini를 사용하는 경우

```env
LLM_PROVIDER=gemini
GEMINI_API_KEY=your_gemini_api_key
GEMINI_MODEL=gemini-1.5-flash
```

### OpenAI를 사용하는 경우

```env
LLM_PROVIDER=openai
OPENAI_API_KEY=your_openai_api_key
OPENAI_MODEL=gpt-4o-mini
```

### Anthropic을 사용하는 경우

```env
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=your_anthropic_api_key
ANTHROPIC_MODEL=claude-3-5-haiku-latest
```

### NEIS API를 사용하는 경우

```env
NEIS_API_KEY=your_neis_api_key
```

`NEIS_API_KEY`는 학교 검색과 학사일정 조회에 사용됩니다. 키가 없거나 호출이 실패하면 일부 화면은 mock 데이터 또는 빈 결과로 동작할 수 있습니다.

전체 예시는 `.env.local.example`을 참고하세요.

## 5. 개발 서버 실행

```powershell
npm run dev
```

정상 실행되면 브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:3000
```

다른 포트로 실행하려면 다음처럼 실행합니다.

```powershell
npm run dev -- -p 3001
```

## 6. 주요 검증 경로

| 경로 | 확인 내용 |
| --- | --- |
| `/onboarding` | 학교 검색 및 프로필 저장 |
| `/dashboard` | 저장된 기록 요약 표시 |
| `/check-in` | 마음 온도 저장 |
| `/temperature` | 온도 그래프 표시 |
| `/complaints/new` | 민원 기록 작성, NEIS 학사일정 표시 |
| `/ai-guide` | 민원 대응 가이드 생성 |
| `/report` | 주간 리포트 생성 |
| `/care` | 정서 케어 추천 생성 |
| `/profile` | 저장된 내 정보 확인 |

## 7. 린트와 빌드

```powershell
npm run lint
npm run build
```

빌드 결과를 실행하려면:

```powershell
npm run start
```

## 8. API 직접 테스트 예시

개발 서버가 실행 중인 상태에서 다른 터미널을 열고 실행합니다.

### 학교 검색

```powershell
curl -X POST http://localhost:3000/api/neis/school-search `
  -H "Content-Type: application/json" `
  -d "{\"schoolName\":\"해솔\",\"schoolLevel\":\"중학교\"}"
```

### 학사일정 조회

```powershell
curl -X POST http://localhost:3000/api/neis/schedule `
  -H "Content-Type: application/json" `
  -d "{\"educationOfficeCode\":\"B10\",\"schoolCode\":\"7132131\",\"date\":\"20260530\"}"
```

## 9. 자주 생기는 문제

### `.env.local`을 수정했는데 반영되지 않음

개발 서버를 재시작해야 합니다.

```powershell
Ctrl + C
npm run dev
```

### 3000 포트가 이미 사용 중임

```powershell
npm run dev -- -p 3001
```

그 다음 `http://localhost:3001`로 접속합니다.

### AI 응답이 예시 데이터로 나옴

- `.env.local`의 `LLM_PROVIDER`가 `mock`인지 확인합니다.
- 실제 provider를 쓰는 경우 API 키가 설정되어 있는지 확인합니다.
- API 키를 수정했다면 개발 서버를 재시작합니다.

### 학교 검색 결과가 비어 있음

- 학교명은 최소 2글자 이상 입력합니다.
- `NEIS_API_KEY`가 필요할 수 있습니다.
- 학교급 필터를 바꿔 다시 검색합니다.
