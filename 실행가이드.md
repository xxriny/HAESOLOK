## 📁 STEP 1 — 프로젝트 폴더로 이동

```powershell
cd "C:\Users\user\Desktop\공공데이터공모전\HAESOLOK"
```

## 📦 STEP 2 — 패키지 설치

```powershell
npm install
```

> ⏳ 최초 1회만 실행하면 됩니다. (`node_modules` 폴더가 생성됩니다)

---

## 🔑 STEP 3 — 환경 변수 설정

프로젝트 루트에 `.env.local` 파일을 생성합니다.

### 방법 A — API 키 없이 바로 실행 (Mock 모드, 추천 ⭐)

`.env.local` 파일을 만들고 아래 내용을 붙여넣기:

```env
LLM_PROVIDER=mock
```

> Mock 모드는 AI API 키 없이 시연용 더미 데이터로 모든 기능을 체험할 수 있습니다.

---

### 방법 B — 실제 AI API 연동

사용할 AI 제공자에 맞게 `.env.local` 작성:

#### 🔷 Gemini (Google) 사용 시
```env
LLM_PROVIDER=gemini
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-1.5-flash
```
> API 키 발급: https://aistudio.google.com/app/apikey

#### 🟢 OpenAI 사용 시
```env
LLM_PROVIDER=openai
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-4o-mini
```

#### 🟠 Anthropic (Claude) 사용 시
```env
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=your_anthropic_api_key_here
ANTHROPIC_MODEL=claude-3-5-haiku-latest
```

---

## 🚀 STEP 4 — 개발 서버 실행

```powershell
npm run dev
```

서버가 시작되면 터미널에 아래와 같이 표시됩니다:

```
▲ Next.js 14.x.x
- Local: http://localhost:3000
```

브라우저에서 👉 **http://localhost:3000** 접속!

## ❓ 자주 발생하는 문제

### Q. `npm install` 중 오류가 발생해요
```powershell
# npm 캐시 초기화 후 재시도
npm cache clean --force
npm install
```

### Q. 포트 3000이 이미 사용 중이에요
```powershell
# 다른 포트로 실행
npm run dev -- -p 3001
```
그 후 http://localhost:3001 으로 접속

### Q. `.env.local` 수정 후 반영이 안 돼요
개발 서버를 **재시작**해야 환경 변수가 적용됩니다.  
터미널에서 `Ctrl + C` → `npm run dev` 재실행
